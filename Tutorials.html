Interaction Demo

The Interaction Demo Project is structured into **Actors, Character, Components, and Interfaces**, with each system handling **object interactions, player controls, and interactive components**. Players can interact with various objects through a **modular system**, ensuring **flexibility and easy expansion**. The **Character System** manages **player movement and interaction logic**, while **Components** enable **dynamic object behavior**.

Interactive Objects System ▲

Manages dynamic objects such as movable cubes and sliding doors.

CubeToInteract (CubeToInteract.h / .cpp)
  • Defines a basic interactable object.
  • Implements Interact() function, likely toggling visibility or physics.
MovingCubeActor (MovingCubeActor.h / .cpp)
  • Handles a cube that moves when interacted with.
  • Uses Tick() to update position over time.
  • Likely moves via SetActorLocation() when activated.

Player Character System ▲

Manages player movement and interaction handling.

MyCharacter (MyCharacter.h / .cpp)
  • Controls movement and camera setup.
  • Implements an interaction system by calling InteractionComponent->TryInteract().
  • Uses BindAction() for key-based interactions.

Interaction Component System ▲

Handles interaction logic for multiple objects.

InteractionComponent (InteractionComponent.h / .cpp)
  • Uses LineTraceByChannel() to detect nearby interactable objects.
  • Calls Object->Interact() if a valid object is found.
  • Likely triggers UI elements (e.g., press "E" to interact).

Interface System ▲

Standardizes interactions for multiple objects.

ObjectInterface (ObjectInterface.h / .cpp)
  • Declares Interact() as a virtual function, allowing any object to be interactable.
  • Implemented in CubeToInteract, MovingCubeActor, and SlidingDoor.
